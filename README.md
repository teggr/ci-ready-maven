# CI Ready Maven

A catalogue of Maven plugins and utilities for building robust CI pipelines and quality gates in enterprise Java projects.

Live site: **https://teggr.github.io/ci-ready-maven/**

## What's In Here

Each tool is documented as a Markdown file with YAML front matter containing:

- `name` – display name
- `category` – e.g. Static Analysis, Coverage, Dependency Governance
- `type` – e.g. Maven Plugin, CLI Tool
- `mavenCoordinates` – e.g. `org.jacoco:jacoco-maven-plugin`
- `lastRelease` – latest known version
- `learnMoreHref` / `learnMoreText` – link to official docs
- `tags` – used for filtering on the site
- `dateAdded` – ISO date

The body of each file is a description followed by a `## Code Example` section with a ready-to-use Maven configuration snippet.

## Building the Site

The site is generated by a [JBang](https://www.jbang.dev/) script (`build.java`) that reads the Markdown files, parses their YAML front matter, and generates a static HTML site in the `_site` directory.

```bash
jbang run build.java
```

The site is then served from the `_site` folder via GitHub Pages. GitHub Actions automatically builds and deploys the site on every push to `main`.

## Contributing

To add a new Maven tool, create a new Markdown file in the root of the repository following the pattern of the existing files. Open a pull request and the site will be rebuilt automatically.
